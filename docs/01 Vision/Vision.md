# 🌟 Vision

## One line

TechEngine is a client/server 3D game engine with a modern, data-oriented core
and a physically-grounded renderer — built solo, to a professional standard,
with the long-term intent of becoming a company.

## Why it exists

- To own a serious piece of technology end-to-end (rendering, ECS, tooling,
  networking) rather than assembling one from black boxes.
- To be the vehicle for deep graphics work (atmospherics, volumetrics, GI).
- To eventually support a product / studio — not just a portfolio piece.

## What "professional scope" means here

- Every major decision is written down (ADR) before it becomes load-bearing.
- Work flows through planning → sprint → implementation → demo → retrospective.
- The renderer is organized as an explicit **render graph** of passes, not an
  ad-hoc call sequence.
- Documentation is a first-class deliverable, not an afterthought.

## Three-year horizon

1. **Foundation** — a stable, well-architected core + render graph you trust.
2. **Signature features** — atmospheric scattering, volumetric fog & shadows,
   a rendering look that is recognizably "TechEngine."
3. **A demo** that proves the engine, plus the docs and process of a studio.

## Non-goals (for now)

- Not chasing feature parity with Unreal/Unity — chasing a *deep, correct* subset.
- Not shipping a marketplace / plugin ecosystem yet.
- Not multi-platform sprawl before the core is solid.
