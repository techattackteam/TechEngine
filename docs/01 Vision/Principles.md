# 🧭 Engineering Principles

Guardrails for decisions. When a choice is hard, these break the tie.

1. **Decide once, write it down.** Load-bearing decisions get an [[ADR Index|ADR]].
   If it's not written, it's not decided.
2. **Refactor by default, rewrite only with evidence.** A rewrite must be
   justified by concrete, documented pain — not by "it feels messy." See
   Sprint 1's assessment.
3. **Architecture over autocomplete.** Highest leverage is in the design, not in
   generating code faster. Use the [[Technical Lead Charter|AI as a technical lead]].
4. **Explicit over implicit.** The render graph, resource lifetimes, and system
   ordering should be visible and inspectable, not emergent side effects.
5. **One session, one task.** Tasks are sized to fit a single coding session
   (~2–6h). If it doesn't fit, split it.
6. **Sustainable pace wins.** The plan is measured in years. Protect rest,
   karting, and the day job. Burnout ends the project faster than any bug.
7. **Demo or it didn't happen.** Every feature ends in a short recorded demo and
   a doc update.
